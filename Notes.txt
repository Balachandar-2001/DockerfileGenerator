💡 Root Cause

After kubeadm init, Kubernetes generates a config file:

$HOME/.kube/config

But in your case, kubectl is NOT using it properly.

So it defaults to:

http://localhost:8080

which is WRONG.

🔵 FIX (MASTER ONLY)

Run these commands exactly:

Step 1: Create kube config directory
mkdir -p $HOME/.kube
Step 2: Copy admin config
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
Step 3: Fix permissions
sudo chown $(id -u):$(id -g) $HOME/.kube/config
Step 4: Test cluster connection
kubectl get nodes

dockergen-master     NotReady
